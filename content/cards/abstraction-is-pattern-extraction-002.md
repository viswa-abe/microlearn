## Idea
When you notice the same structure in two different situations, and you name that structure -- that's abstraction. It's not about making things vague. It's about **extracting the pattern that two concrete things share**.

## Example
```
Sorting a deck of cards
Sorting names alphabetically
Sorting files by date
```
The concrete details differ completely. But the pattern -- *compare two things, swap if out of order, repeat* -- is identical. Naming that pattern "sorting" is abstraction.

## Probe
If abstraction is pattern extraction, then a *bad* abstraction is one that groups things by surface similarity rather than shared structure. Can you think of a category that feels natural but groups things that actually work differently?
